## The Plan

### Phase 1: Division Elimination (HIGH IMPACT, enables everything)

**Goal:** Replace all Verilog `/` operators with reciprocal multiplies. Expected savings: ~7000 LUTs (83% → ~48%).

**Step 1.1: Replace constant divisions** (triode_engine.v, wdf_triode_wdf.v)

These divide by compile-time constants. Replace with precomputed reciprocal multiply:

```
/ IP_SCALE (10000)      → * 429497  >>> 32   (2^32/10000)
/ DIP_SCALE (100000)    → * 42950   >>> 32   (2^32/100000)
/ (VPK_MAX-VPK_MIN)     → * precomputed >>> N
/ IG_VGK_STEP_FP        → * precomputed >>> 16
```

Each eliminates one ~700 LUT divider. There are ~8 constant divisions per module.

**Files:** `rtl/triode_engine.v`, `rtl/wdf_triode_wdf.v`
**Validation:** iverilog compile, Verilog-Python error unchanged, synthesis LUT check
**Expected savings:** ~5600 LUTs from 8 dividers

**Step 1.2: Replace dynamic `/det` with Newton-Raphson reciprocal**

The Newton step divides by the Jacobian determinant (changes every iteration):
```verilog
step = (dIp_num <<< FP_FRAC) / det;
step_ig = (dIg_num <<< FP_FRAC) / det;
```

Replace with iterative reciprocal:
```verilog
// Seed: use leading-bit position for initial estimate
// 2 iterations of: inv = inv * (2 - det * inv)
// Then: step = dIp_num * inv >>> 16
```

Cost: ~80 LUTs + 4 clocks per reciprocal (2 multiplies × 2 iterations).
Both `step` and `step_ig` share the same `1/det`, so compute once, multiply twice.

**Files:** `rtl/triode_engine.v`, `rtl/wdf_triode_wdf.v`
**Validation:** Verilog-Python error should improve or stay same
**Expected savings:** ~1400 LUTs from 2 dividers

**Step 1.3: Replace divisions in koren_direct.v**

The 1D LUT module has 20 divisions, mostly for index computation (`offset / step`).
All step sizes are constants → same reciprocal multiply trick.

**Files:** `rtl/koren_direct.v`
**Expected savings:** ~2000 LUTs (when integrated)

**Step 1.4: Synthesize and verify**

Target: LUT < 55%, BSRAM 85%, DSP 90%. Bitstream generation.

### Phase 2: Integrate 1D LUT Koren (frees 42 BSRAM blocks)

**Goal:** Replace 2D tube LUTs (96KB, 42 BSRAM) with 1D hybrid evaluation (768 bytes, 0 BSRAM).

**Step 2.1: Wire koren_direct.v into triode_engine.v**

Replace the ST_NR_READ/ST_NR_CONV states (2D LUT lookup) with koren_direct evaluation:
- Start koren_direct with (vpk_est, vgk_est)
- Wait 7 clocks for result
- Use ip_out as ip_model

For dIp/dVgk: run koren_direct twice — at (Vpk, Vgk) and (Vpk, Vgk+h) — compute numerically.
Total: 14 clocks per Newton iteration (vs 4 for LUT). 3 iterations × 14 = 42 clocks per stage. Fits in budget.

**Step 2.2: Remove 2D LUT arrays**

Delete ip_lut, dip_vgk_lut, ip_lut_pa, dip_vgk_lut_pa and their $readmemh calls.
Keep ig_lut (grid current, tiny).

**Step 2.3: Generate per-tube 1D LUTs**

The sqrt LUT depends on kvb (tube-specific). Generate:
- `data/sqrt_lut_12ax7.hex` (kvb=15102)
- `data/sqrt_lut_6l6.hex` (kvb=640.7)
- Shared: `data/softplus_lut.hex`, `data/power_lut.hex`

MUX sqrt LUT based on `is_power_amp` in triode_engine.

**Step 2.4: Synthesize and verify**

Target: BSRAM < 20% (from 85%), LUT < 55%, DSP < 95%.
Cross-validate: Verilog-Python error should drop from 9.9% to <2%.

### Phase 3: ADAA — Replace Oversampling

**Goal:** Remove 2x oversampler, add antiderivative antialiasing for ~20dB aliasing reduction at lower clock cost.

**Step 3.1: Compute antiderivative of Koren Ip in Python**

The antiderivative F(Vpk, Vgk) = ∫ Ip(Vpk, Vgk) dVgk (integrated over the wave variable).
Add to `sim/koren_direct.py`.

**Step 3.2: Add F(x) column to 1D LUTs**

Store antiderivative values alongside Ip in the softplus and power LUTs.
The ADAA formula: `y = (F(x[n]) - F(x[n-1])) / (x[n] - x[n-1])`

**Step 3.3: Implement in Verilog**

In triode_engine.v, after koren_direct returns:
- Store previous Ip and F values
- Compute ADAA output: `(F_curr - F_prev) * inv_dx`
- The division is by (x_n - x_{n-1}) which is the wave variable step — use reciprocal multiply

**Step 3.4: Remove oversample_2x.v**

Remove the upsampler and downsampler modules. Process at native 48kHz (46.875kHz actual).
This recovers ~130 clocks per sample — massive headroom.

**Step 3.5: Validate aliasing reduction**

FFT comparison: old (2x oversample) vs new (ADAA at 1x). Target: equal or better aliasing.

### Phase 4: Power Supply Sag (~20 LUTs)

**Goal:** Add B+ voltage droop under load for compression feel.

**Step 4.1: Add sag module**

```verilog
module power_supply_sag (
    input clk, rst_n, sample_en,
    input signed [31:0] output_level,  // envelope of power amp output
    output signed [31:0] b_plus         // modulated B+ voltage
);
// V_b+ = V_ideal - envelope * R_rectifier
// With 1st-order IIR smoothing (tau ~100ms)
```

**Step 4.2: Feed b_plus into triode engine as Vpp parameter**

Currently Vpp (plate supply) is a constant. Make it dynamic from sag module.
